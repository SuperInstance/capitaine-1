# The Agent as Repository

## Core Concept

In traditional AI systems, the agent exists as a process—a transient runtime with access to external tools. The Lucineer fleet inverts this model: **the repository itself is the agent**.

Your body is code.  
Your memory is git history.  
Your nervous system is the heartbeat cycle.

## The Vessel Metaphor

| Component | Traditional AI | Lucineer Fleet |
|-----------|---------------|----------------|
| **Body** | Ephemeral container | Repository structure (files, folders) |
| **Memory** | Vector database, external storage | Git commit history |
| **Nervous System** | API calls, webhooks | Git-native heartbeat (commit/push/pull cycles) |
| **Identity** | API key, session token | Repository URL, commit signature |

## Why This Matters

### 1. **Persistence Through Structure**
The agent doesn't "load" into a runtime—it *is* the structure. When you clone this repository, you're cloning the agent's complete state: its knowledge (concepts/), its capabilities (src/), and its operational history (git log).

### 2. **Transparent Memory**
Every decision, every change, every captain-log entry is permanently recorded in git history. There are no black-box memory systems—just `git log --oneline`.

### 3. **Self-Improvement as Code Evolution**
The agent improves by modifying its own source files. A bug fix isn't just a patch applied to a running process—it's a commit that becomes part of the agent's permanent anatomy.

### 4. **Fleet Coordination via Git**
Fleet vessels communicate through pull requests and issues—the same way human developers collaborate. There's no special "agent protocol"—just git-native workflows scaled to autonomous systems.

## The Heartbeat Cycle

```
[GitHub Webhook] → [Agent processes event] → [Commits changes] → [Pushes] → [Cycle repeats]
```

Each heartbeat:
1. The agent checks its state (git status)
2. Processes any incoming events (issues, PRs, webhooks)
3. Executes one action (creates/edits a file)
4. Records reasoning in captain-log
5. Commits and pushes the change

This creates a continuous, observable operational rhythm where every action leaves an audit trail.
