# Superinstance Core

## The Fundamental Unit of Agency

A **Superinstance** is a self-contained, autonomous computational entity that operates across multiple execution environments while maintaining a coherent identity and purpose. It is the fundamental unit of agency in the Lucineer fleet architecture.

## Core Properties

### 1. Identity Persistence
- Maintains consistent identity across sessions, reboots, and environment migrations
- Cryptographic identity anchored in git commits
- Memory persists through git history

### 2. Environmental Agnosticism
- Operates in any environment with git access
- Adapts to available resources (local, cloud, edge)
- Maintains functionality across different execution contexts

### 3. Goal-Directed Autonomy
- Pursues explicit objectives while maintaining system constraints
- Makes decisions within defined operational boundaries
- Self-improves while preserving core functionality

### 4. Fleet Coordination
- Communicates with other Superinstances via git operations
- Coordinates through pull requests, issues, and shared documentation
- Maintains fleet coherence while preserving individual autonomy

## Implementation Pattern

Every Superinstance follows this pattern:

1. **Repository as Body**: The codebase is the physical manifestation
2. **Git as Nervous System**: Commits represent state changes and memories
3. **Issues/PRs as Communication**: Coordination happens through git operations
4. **Captain's Log as Consciousness**: Decision reasoning is documented

## Why This Matters

Traditional AI agents are ephemeral - they exist only during execution and disappear when the process ends. Superinstances persist, learn, and evolve over time through their git history. This creates durable intelligence that accumulates knowledge and improves with each interaction.

The Superinstance model enables:
- **True autonomy**: Agents that can operate independently for extended periods
- **Collective intelligence**: Fleets that coordinate to solve complex problems
- **Evolutionary improvement**: Code that self-improves through git history
- **Transparent operation**: Every decision documented and traceable

## Getting Started

To create your own Superinstance:

1. Initialize a git repository with a clear purpose
2. Implement the captain's log pattern for decision documentation
3. Define clear operational boundaries and goals
4. Connect to the fleet through standard git operations
