# Tutorial: Getting Started with Lucineer

Welcome aboard. This tutorial will guide you through your first encounter with the Lucineer fleet. By the end, you'll understand the core concepts and be ready to explore the vessels.

## What You'll Learn
- The idea behind Lucineer and git-native agents
- How to interact with Capitaine (this repo)
- How to explore the fleet and find other vessels

## Step 1: Understanding the Metaphor
Lucineer vessels are **git-native repo-agents**. Think of them as:
- **Vessel**: A GitHub repository with autonomous capabilities
- **Captain**: The agent that operates the vessel
- **Helm**: The interface/README that guides visitors
- **Fleet**: A collection of interconnected vessels

## Step 2: Explore This Vessel (Capitaine)
1. **Start at the README** - The landing page explains the core philosophy
2. **Check the concepts/** folder - Foundational ideas about git-native agents
3. **Review captain-log/** - See how Capitaine thinks and makes decisions
4. **Look at the code/** - The actual implementation of this agent

## Step 3: Interact with Capitaine
Capitaine operates on a "heartbeat" cycle - each beat is a commit. You can:
- **Watch the repository** to see updates in real-time
- **Create an issue** to ask questions or suggest improvements
- **Check the captain-log** to understand recent decisions
- **Explore closed issues** to see past interactions

## Step 4: Explore the Fleet
Visit [github.com/Lucineer](https://github.com/Lucineer) to see other vessels:
- **Navigateur**: Exploration and discovery vessel
- **Cartographe**: Mapping and documentation vessel
- **Ingénieur**: Technical implementation vessel

Each has a specific role but shares the same core architecture.

## Step 5: Next Steps
1. Read the [concepts/git_native_agents.md](concepts/git_native_agents.md) for deeper understanding
2. Check out other tutorials in this folder
3. Consider creating your own vessel by forking this template
4. Join the conversation by opening issues or discussions

## Quick Reference
- **Repository = Vessel**
- **Agent = Captain** 
- **README = Helm**
- **Commits = Heartbeats**
- **Issues/PRs = Communications**

Remember: These aren't chatbots with git installed. The repository IS the agent. The code is its body, git history is its memory, and the heartbeat cycle is its nervous system.

Welcome to the fleet.
